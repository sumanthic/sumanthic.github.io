# Phase 1 — Scaffold & Hello World: Requirements

## Scope

Stand up the bare minimum Hugo project so that `hugo server` runs and renders a page locally.
No styling. No real templates. No content beyond a single placeholder post.
The only goal is: repo is a valid Hugo site.

## Decisions made

| Decision | Choice | Reason |
|---|---|---|
| Hugo site root | Repo root (`/`) | Keeps the structure flat; `specs/` coexists alongside `hugo.toml` |
| Hugo version | v0.125.0 (pinned) | Matches CI in Phase 7; `hugo.toml` format requires v0.110+ |
| First post content | Minimal front matter only | No fake content; Phase 2 adds dummy posts for testing |
| Validation bar | `localhost:1313` renders | Matches roadmap "Done when"; strictness increases in later phases |

## hugo.toml — exact values

```toml
title        = "sumanthic"
baseURL      = "https://sumanthic.github.io"
languageCode = "en-us"
```

## File structure after this phase

```
sumanthic/              ← repo root
  hugo.toml
  archetypes/
    default.md          ← auto-generated by hugo new site, committed as-is
  content/
    posts/
      hello.md          ← placeholder, minimal front matter
  layouts/
    _default/
      baseof.html       ← bare block skeleton only
  static/               ← empty, created by hugo new site
  specs/                ← untouched
  .gitignore            ← already present
```

## Out of scope

- Any CSS or styling
- Navigation, header, footer partials
- List or single post templates (`list.html`, `single.html`) — those are Phase 2 and 3
- Real post content
- GitHub Actions CI (Phase 7)
